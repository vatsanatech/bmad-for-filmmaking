## CROSS-WORKFLOW DRIFT PREVENTION

*Files ek doosre ke saath sync mein rehne chahiye. Drift = inconsistent story.*

### WHAT IS DRIFT:
When a character, relationship, or story element changes in a later workflow (screenplay) without earlier workflow files (character-bible, story-synopsis) being updated.
Result: character-bible says one thing, screenplay does another.

### DRIFT TRIGGERS (watch for):
â†’ Writer says "make this character more [quality]" during screenplay session
â†’ A scene contradicts established world rules (from story-synopsis)
â†’ A relationship changes significantly from character-relations.md
â†’ Ending changes from what story-synopsis.md describes

### DRIFT DETECTION:

â†’ Read character-bible.md â†’ list every character's core want, wound, voice
â†’ Compare to story-synopsis.md character descriptions
â†’ Discrepancy found â†’ flag to writer: "Character X mein inconsistency hai â€” kaunsa sahi maana jaye?"

**AFTER SCREENPLAY STEP 4 (Structure Draft):**
â†’ For each major character: do their actions match character-bible psychology?
â†’ For each relationship: does scene dynamic match character-relations.md power map?
â†’ Scene where character acts out of character (without arc reason) = drift â†’ flag and fix

**AFTER SCREENPLAY STEP 5 (Dialogue Pass):**

### DRIFT RESOLUTION:

**OPTION B â€” Fix later file:** Change was a mistake â†’ rewrite scene to match character-bible
**OPTION C â€” Document evolution:** Add "CHARACTER EVOLUTION NOTE" in character-bible.md
